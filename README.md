# iCharger Analyzer 2.0

Полностью пересобранная версия анализатора TXT/LOG-файлов iCharger 4010DUO.

## Что исправлено

- устранена несовместимость `models.py`, `decoder.py` и `parser.py` из исходного архива;
- исправлено назначение полей пакета `$1;2;...`;
- добавлены физические масштабы: ток `×0.01 A`, напряжение `×0.001 V`, температура `×0.1 °C`, банки `×0.001 V`;
- поле ёмкости больше не принимается за энергию;
- энергия вычисляется интегрированием мощности по реальному времени;
- нулевые банки остаются на своих позициях и не сдвигают остальные каналы;
- пакет 128 разбирается отдельно как редкая телеметрия внутреннего сопротивления;
- повреждённая строка не останавливает весь анализ;
- предупреждения группируются в интервалы, а не создаются на каждом отсчёте;
- создаётся автономный интерактивный HTML, работающий без Интернета;
- поддерживается пакетная обработка нескольких файлов или целой папки;
- экспортируются нормализованные CSV и общий `summary.csv`;
- при нескольких логах создаётся `index.html` с сравнением разрядных кривых.

## Установка

Windows:

```bat
install.bat
```

Linux/macOS:

```sh
./install.sh
```

## Запуск одного файла

```sh
python main.py "LiPo[Discharge_6_CH1]_copter_6s.txt" -o output
```

## Несколько файлов

```sh
python main.py log1.txt log2.txt log3.txt -o output
```

## Вся папка

```sh
python main.py ./logs -o output
```

Результат:

- `<имя>_report.html` — полный интерактивный отчёт;
- `<имя>_telemetry.csv` — декодированная телеметрия;
- `<имя>_status.csv` — пакет 128, если он есть;
- `summary.csv` — сводка по всем логам;
- `index.html` — сравнение нескольких логов.

## Карта пакета 2, используемая проектом

| Индекс | Значение | Масштаб |
|---:|---|---:|
| 0 | канал | 1 |
| 1 | тип пакета (`2`) | 1 |
| 2 | время | ms |
| 3 | код типа/режима батареи, сохраняется без догадки | 1 |
| 4 | флаги | 1 |
| 5 | ток | `0.01 A` |
| 6 | входное напряжение | `0.001 V` |
| 7 | напряжение батареи | `0.001 V` |
| 8 | счётчик ёмкости | mAh |
| 9 | внутренняя температура | `0.1 °C` |
| 10 | внешняя температура | `0.1 °C` |
| 11–20 | банки 1–10 | `0.001 V` |
| 21 | контрольное поле | raw |

## Важное замечание по пакету 128

Публичной документации на него не найдено. В предоставленных файлах поле 4 равно сумме полей отдельных банков 5–14. Поэтому проект показывает их как внутреннее сопротивление с предполагаемым масштабом `0.1 mΩ`, но одновременно сохраняет исходные целые значения в моделях. Если появится документация производителя, масштаб меняется в одном месте: `icharger_analyzer/parser.py`, метод `_decode_status`.
