# 🔋 iCharger Analyzer

### Интерактивные графики, статистика и диагностика для TXT/LOG-файлов iCharger

Преобразует сырые журналы iCharger в наглядные автономные HTML-отчёты, нормализованные CSV-файлы и сравнительные разрядные кривые.

**Поддерживает одиночные файлы, несколько логов и обработку целой папки.**

## Возможности

| Функция | Что делает |
|---|---|
| 📈 Интерактивные графики | Масштабирование, перемещение, общая подсказка по времени и включение/выключение отдельных кривых |
| 🔌 Автономный HTML | По умолчанию Plotly встраивается в отчёт, поэтому Интернет для просмотра не нужен |
| 🔋 Анализ батареи | Напряжение пакета, ток, мощность, ёмкость, энергия, температура и напряжения отдельных банок |
| ⚖️ Контроль разбаланса | Минимальная, максимальная и средняя банка, а также разница между банками в mV |
| 🧮 Корректное интегрирование | Ёмкость и энергия рассчитываются по фактическим временным интервалам между отсчётами |
| 🚨 Диагностика | Группирует опасные или подозрительные участки в понятные временные события |
| 📂 Пакетная обработка | Обрабатывает один файл, несколько файлов или все `.txt`/`.log` в каталоге |
| 🔍 Устойчивый парсер | Повреждённая строка фиксируется в отчёте, но не останавливает обработку всего файла |
| 📊 Сравнение испытаний | Для нескольких логов создаёт общий `index.html` с разрядными кривыми |
| 🧾 Экспорт CSV | Сохраняет все декодированные и вычисленные значения для Excel, MATLAB, Python и других систем |

## Быстрый запуск

### Windows

1. Распакуйте проект.
2. Откройте папку проекта.
3. Запустите:

```bat
install.bat
```

4. Перетащите лог в папку проекта или укажите полный путь:

```bat
run.bat "C:\iCharger\Logs\LiPo[Discharge_6_CH1]_copter_6s.txt"
```

5. Откройте созданный файл:

```text
output\<имя_лога>_report.html
```

### Linux / macOS

```bash
chmod +x install.sh run.sh
./install.sh
./run.sh "/path/to/log.txt"
```

Готовый отчёт появится в каталоге `output`.

## Использование

Общий синтаксис:

```bash
python main.py <файл_или_папка> [ещё_файлы] -o <папка_результата>
```

### Один лог

```bash
python main.py "LiPo[Discharge_6_CH1]_copter_6s.txt" -o output
```

### Несколько логов

```bash
python main.py \
  "battery_01.txt" \
  "battery_02.txt" \
  "battery_03.log" \
  -o output
```

Для Windows `cmd` можно записать в одну строку:

```bat
python main.py "battery_01.txt" "battery_02.txt" "battery_03.log" -o output
```

### Вся папка

```bash
python main.py "./logs" -o output
```

Из указанной папки будут взяты файлы с расширениями:

```text
.txt
.log
```

Обход выполняется только по самой указанной папке, без рекурсивного поиска во вложенных каталогах.

### Через готовые скрипты

Windows:

```bat
run.bat "C:\iCharger\Logs"
```

Linux/macOS:

```bash
./run.sh "/home/user/icharger/logs"
```

Скрипты автоматически используют каталог `output`.

## Что создаётся после обработки

Для каждого входного файла:

| Файл | Назначение |
|---|---|
| `<имя>_report.html` | Полный интерактивный отчёт |
| `<имя>_telemetry.csv` | Основная телеметрия и вычисленные параметры |
| `<имя>_status.csv` | Данные пакета 128, если он присутствует |

Общие файлы:

| Файл | Когда создаётся |
|---|---|
| `summary.csv` | После обработки хотя бы одного исправного лога |
| `index.html` | Когда успешно обработано два или более логов |

Пример структуры результата:

```text
output/
├── index.html
├── summary.csv
├── LiPo_Discharge_6_CH1_copter_6s_report.html
├── LiPo_Discharge_6_CH1_copter_6s_telemetry.csv
├── LiPo_Discharge_6_CH1_copter_6s_status.csv
├── LiIo_Discharge_30_CH1_m5_5s_19Ah_report.html
└── LiIo_Discharge_30_CH1_m5_5s_19Ah_telemetry.csv
```

> Символы, непригодные для имени выходного файла, автоматически заменяются на `_`. Исходный лог не изменяется.

## Что отображается в HTML-отчёте

### Сводные показатели

- длительность испытания;
- количество отсчётов;
- медианный период записи;
- количество обнаруженных балансирных каналов;
- начальное, конечное, минимальное и максимальное напряжение пакета;
- пиковый ток заряда и разряда;
- средний ток;
- максимальная абсолютная мощность;
- ёмкость по счётчику iCharger;
- ёмкость по интегралу тока;
- энергия по интегралу мощности;
- минимальное и максимальное напряжение банки;
- максимальный и конечный разбаланс;
- максимальная внутренняя и внешняя температура;
- последние доступные значения внутреннего сопротивления;
- количество повреждённых строк и пакетов 128.

### Временные графики

1. напряжение аккумулятора и входное напряжение зарядного устройства;
2. ток и мощность;
3. ёмкость и накопленная энергия;
4. внутренняя и внешняя температура;
5. напряжение каждой доступной банки;
6. разброс между минимальной и максимальной банкой;
7. внутреннее сопротивление из пакета 128, когда такие данные присутствуют.

### Разрядные кривые

- напряжение пакета от отданной ёмкости;
- напряжения отдельных банок от отданной ёмкости.

### Дополнительная диагностика

Отчёт также содержит:

- таблицу сгруппированных предупреждений;
- метаданные зарядного устройства;
- диагностические сведения о повреждённых строках;
- JSON-сводку с рассчитанными показателями.

## Параметры командной строки

Посмотреть встроенную справку:

```bash
python main.py --help
```

Доступные параметры:

| Параметр | Назначение |
|---|---|
| `inputs` | Один или несколько TXT/LOG-файлов либо каталогов |
| `-o PATH` | Каталог выходных файлов; по умолчанию `output` |
| `--output PATH` | Полная форма параметра `-o` |
| `--cdn` | Подключать Plotly из CDN вместо встраивания библиотеки в HTML |
| `--no-csv` | Не создавать декодированные CSV-файлы |
| `-h`, `--help` | Показать справку |

### Автономный HTML или CDN

По умолчанию создаётся **автономный отчёт**:

```bash
python main.py log.txt -o output
```

Преимущества:

- работает без Интернета;
- удобно архивировать вместе с испытанием;
- открывается на другом компьютере без установки Python.

Недостаток: HTML-файл получается больше.

Для уменьшения размера отчёта:

```bash
python main.py log.txt -o output --cdn
```

В этом режиме для открытия графиков нужен доступ к CDN Plotly.

### Только HTML без CSV

```bash
python main.py log.txt -o output --no-csv
```

`summary.csv` при этом всё равно создаётся, поскольку он является общей сводкой запуска.

## Автоматические предупреждения

Предупреждения не создаются для каждого отдельного отсчёта. Соседние срабатывания объединяются в один временной интервал, чтобы отчёт отражал физическое событие, а не заполнялся сотнями одинаковых строк.

| Уровень | Событие | Условие |
|---|---|---|
| Высокий | Высокая внутренняя температура | `≥ 60 °C` |
| Высокий | Сильный разбаланс банок | `Δ ≥ 50 mV` |
| Высокий | Низкое напряжение банки | минимальная банка `≤ 3.00 V` |
| Средний | Заметный разбаланс банок | `20 mV ≤ Δ < 50 mV` |
| Средний | Разрыв телеметрии | интервал значительно превышает типичный период записи |

> Пороги являются диагностическими настройками программы, а не универсальными безопасными пределами для любой химии аккумуляторов. Решение о допустимых напряжениях и температурах должно учитывать тип элемента, документацию производителя и режим испытания.

## Формат данных iCharger

Проект рассчитан на журналы с разделителем `;`, создаваемые iCharger 4010DUO и совместимые с наблюдаемой структурой пакетов.

Пример заголовка:

```text
@Model:4010DUO; Fireware:V2.20; Hardware:V2.00; SN:2510118006
```

Устройство использует написание `Fireware`; парсер понимает как `Fireware`, так и `Firmware`.

### Основной пакет `$1;2;...`

После удаления начального символа `$` пакет содержит 22 целочисленных поля:

| Индекс | Значение | Масштаб |
|---:|---|---:|
| 0 | Канал | `1` |
| 1 | Тип пакета: `2` | `1` |
| 2 | Время | `ms` |
| 3 | Код типа/режима батареи | сохраняется как raw-код |
| 4 | Флаги | сохраняются как raw |
| 5 | Ток | `× 0.01 A` |
| 6 | Входное напряжение | `× 0.001 V` |
| 7 | Напряжение аккумулятора | `× 0.001 V` |
| 8 | Счётчик ёмкости | `mAh`, знаковое значение |
| 9 | Внутренняя температура | `× 0.1 °C` |
| 10 | Внешняя температура | `× 0.1 °C` |
| 11–20 | Банки 1–10 | `× 0.001 V` |
| 21 | Контрольное поле | raw |

Пример:

```text
$1;2;10000;1;0;-500;24324;14904;-17317;398;0;...
```

Интерпретация основных полей:

```text
ток                 = -5.00 A
входное напряжение  = 24.324 V
напряжение батареи  = 14.904 V
счётчик ёмкости     = -17317 mAh
температура         = 39.8 °C
```

Нулевое значение банка́ового канала интерпретируется как отсутствие измерения и сохраняется как пустое значение. Номера последующих банок при этом **не сдвигаются**.

### Пакет `$1;128;...`

Публичная спецификация пакета 128 отсутствует. В исследованных логах наблюдается устойчивое соотношение:

```text
поле суммы ≈ сумма значений отдельных банков
```

Поэтому проект отображает эти данные как предполагаемое внутреннее сопротивление с масштабом `0.1 mΩ`.

Это **обоснованная, но не официально подтверждённая интерпретация**. Исходные целые значения сохраняются в `raw_values`, поэтому после появления документации производителя декодирование можно скорректировать без изменения основной архитектуры.

Место настройки:

```text
icharger_analyzer/parser.py
IChargerParser._decode_status()
```

### Расчёт ёмкости

Ёмкость рассчитывается трапецеидальным интегрированием тока по реальному времени:

```text
ΔQ = ((I[n] + I[n-1]) / 2) × Δt / 3600
```

### Расчёт энергии

Энергия рассчитывается по мощности:

```text
P = U × I
ΔE = ((P[n] + P[n-1]) / 2) × Δt / 3600
```

В отчёте используется абсолютный поток энергии и ёмкости, поэтому результат удобен как для разряда, так и для заряда.

<details>
<summary><strong>Для 1S указано 0 обнаруженных банок</strong></summary>

Количество банков определяется по ненулевым балансирным каналам. В некоторых 1S-логах зарядное устройство записывает только общее напряжение пакета, а все поля банок оставляет нулевыми.

Такой файл всё равно анализируется: графики напряжения, тока, мощности, ёмкости, энергии и температуры будут построены. Недоступны только графики отдельных банков и разбаланса.

</details>

<details>
<summary><strong>Один из файлов повреждён</strong></summary>

Повреждённые строки добавляются в диагностическую таблицу отчёта. Если в файле остаётся хотя бы один исправный пакет телеметрии, анализ продолжается.

При пакетном запуске ошибка одного файла не останавливает обработку остальных. Команда завершится с ненулевым кодом возврата, а исправные отчёты сохранятся.

</details>

## Ограничения и важные замечания

- Проект проверен на структуре журналов iCharger 4010DUO с пакетами типов `2` и `128`.
- Неизвестные типы пакетов подсчитываются, но не декодируются.
- Контрольное поле сохраняется, но криптографическая или протокольная проверка checksum не выполняется.
- Пакет 128 интерпретируется на основании наблюдаемой структуры, а не официальной спецификации.
- Каталоги обрабатываются без рекурсивного обхода вложенных папок.
- Автоматические предупреждения являются вспомогательной диагностикой, а не заменой требованиям производителя аккумулятора.
- Исходные TXT/LOG-файлы программа не изменяет.
- Для очень крупных журналов автономный HTML может занимать значительный объём, поскольку содержит Plotly и все графические данные.

## Коды завершения

| Код | Значение |
|---:|---|
| `0` | Все входные файлы успешно обработаны |
| `1` | Хотя бы один файл завершился ошибкой, остальные могли быть обработаны |
| `2` | Ошибка входного пути или подходящие логи не найдены |
